This package is responsible to control a mobile robot when receives a {\ttfamily \mbox{\hyperlink{namespacego__to__point}{go\+\_\+to\+\_\+point}}} command from the user.\hypertarget{md__r_e_a_d_m_e_autotoc_md4}{}\doxysection{Description of the system}\label{md__r_e_a_d_m_e_autotoc_md4}
The launch file will open and start\+:
\begin{DoxyEnumerate}
\item The simulation environment
\item The node {\ttfamily random\+\_\+position\+\_\+server}, which implements a random position service
\item The node {\ttfamily \mbox{\hyperlink{namespacego__to__point}{go\+\_\+to\+\_\+point}}}, which implements a service to drive a robot toward a point in the environment
\item The node {\ttfamily state\+\_\+machine}, which implements a service to start or stop the robot, and calls the other two services to drive the robot
\item The node {\ttfamily \mbox{\hyperlink{namespaceuser__interface}{user\+\_\+interface}}}, which asks the user to start/stop the robot, and calls the service implemented in the state\+\_\+machine node
\item The service implemented in the {\ttfamily \mbox{\hyperlink{namespacego__to__point}{go\+\_\+to\+\_\+point}}} node is able to drive the robot towards a certain position in space (x, y) and with a certain angle (theta)
\item The service implemented in the {\ttfamily random\+\_\+position\+\_\+server} node replies with random values for x, y, and theta, where x and y should be limited between some min and max values
\item The service implemented in the robot {\ttfamily state\+\_\+machine} node gives the possibility to start or stop the robot behaviour.
\end{DoxyEnumerate}

Because our user request is implemented as an action, it can be preempted, this way we can stop the robot at any time during the simulation and then restart it with a new goal.
